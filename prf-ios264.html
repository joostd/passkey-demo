Safari 26.4 PRF test

Test support for the WebAuthn PRF extension in Safari 26.4.


Instructions:
  • CREATE a passkey on a FIDO security key
  • If no PRF output appears, GET an assertion
  • Observe the PRF output
  • Change browsers (e.g. Chrome on macOS) and GET an assertion
  • Observe the PRF output, and check it matches the previous one
  • Reload to start over...
With passkeys stored on FIDO security keys, Safari/Webkit on iOS 26.4 and macOS 26.4 has a bug that calculates incorrect PRF output. PRF output is correct however with passkeys stored in iCloud using the iOS/macOS platform authenticator.
